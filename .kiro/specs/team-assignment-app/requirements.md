## Requirements

### 要件1: メンバー管理

**目的:** プロジェクトチームのユーザーとして、メンバーを管理できるようにしたい。そうすることで、チーム分けに必要な参加者情報を準備できる。

#### 受入基準
1. アプリケーションを開いたとき、チーム分けアプリは、ひらがなメンバー5人(あ、い、う、え、お)を自動的に表示すること
2. アプリケーションを開いたとき、チーム分けアプリは、カタカナメンバー5人(ア、イ、ウ、エ、オ)を自動的に表示すること
3. ユーザーがメンバーを追加するとき、チーム分けアプリは、名前の入力を必須とすること
4. ユーザーがメンバーを編集するとき、チーム分けアプリは、名前の変更を可能にすること
5. ユーザーがメンバーを削除するとき、チーム分けアプリは、そのメンバーをリストから除外すること

### 要件2: チーム分けロジック(チーム数指定)

**目的:** プロジェクトチームのユーザーとして、指定したチーム数でメンバーを分けたい。そうすることで、必要なチーム構成を簡単に作成できる。

#### 受入基準
1. ユーザーがチーム数を指定するとき、チーム分けアプリは、1以上の整数値の入力を受け付けること
2. ユーザーがチーム分け実行ボタンをクリックしたとき、チーム分けアプリは、全メンバーを指定されたチーム数にランダムに振り分けること
3. メンバー数がチーム数で割り切れない場合、チーム分けアプリは、余ったメンバーをランダムにいずれかのチームに配分すること
4. チーム分け実行後、チーム分けアプリは、各チームのメンバーリストを即時に表示すること

### 要件3: チーム分けロジック(1チームの人数指定)

**目的:** プロジェクトチームのユーザーとして、1チームあたりの人数を指定してメンバーを分けたい。そうすることで、チームサイズを制御できる。

#### 受入基準
1. ユーザーが1チームの人数を指定するとき、チーム分けアプリは、1以上の整数値の入力を受け付けること
2. ユーザーがチーム分け実行ボタンをクリックしたとき、チーム分けアプリは、指定された人数ごとにメンバーをランダムに振り分けること
3. 全メンバー数が指定人数で割り切れない場合、チーム分けアプリは、余ったメンバーでチームを作成すること
4. チーム分け実行後、チーム分けアプリは、各チームのメンバーリストを即時に表示すること

### 要件4: チーム分け制約(メンバー種別制約)

**目的:** プロジェクトチームのユーザーとして、1チームにひらがなメンバーまたはカタカナメンバーのみを配置する制約を設定したい。そうすることで、特定の条件下でのチーム分けができる。

#### 受入基準
1. ユーザーが制約設定で「1チームのみのひらがなメンバー数」を指定するとき、チーム分けアプリは、0以上の整数値の入力を受け付けること
2. ユーザーが制約設定で「1チームのみのカタカナメンバー数」を指定するとき、チーム分けアプリは、0以上の整数値の入力を受け付けること
3. ユーザーがチーム分け実行ボタンをクリックしたとき、チーム分けアプリは、指定された制約に従って1つのチームを作成すること
4. 制約が指定されている場合、チーム分けアプリは、その制約チームを作成した後、残りのメンバーを他のチームにランダムに振り分けること

### 要件5: 結果表示

**目的:** プロジェクトチームのユーザーとして、チーム分けの結果を視覚的に確認したい。そうすることで、分け方が適切かどうかを判断できる。

#### 受入基準
1. チーム分け実行後、チーム分けアプリは、各チームをリスト形式で表示すること
2. チーム分け実行後、チーム分けアプリは、各チームをカード形式で表示すること
3. チーム分けアプリは、各チームに所属するメンバー名を明確に表示すること
4. チーム分けアプリは、各チームを識別可能にするため、チーム番号またはチーム名を表示すること

### 要件6: 再抽選機能

**目的:** プロジェクトチームのユーザーとして、チーム分けの結果が気に入らない場合に再度実行したい。そうすることで、より満足のいくチーム構成を得られる。

#### 受入基準
1. チーム分け結果表示後、チーム分けアプリは、再抽選ボタンを表示すること
2. ユーザーが再抽選ボタンをクリックしたとき、チーム分けアプリは、同じ設定条件で新しいチーム分けを即座に実行すること
3. 再抽選実行時、チーム分けアプリは、前回の結果を新しい結果で上書きすること

### 要件7: 結果の画像保存

**目的:** プロジェクトチームのユーザーとして、チーム分けの結果を画像として保存したい。そうすることで、結果を簡単に共有・記録できる。

#### 受入基準
1. チーム分け結果表示後、チーム分けアプリは、画像保存ボタンを表示すること
2. ユーザーが画像保存ボタンをクリックしたとき、チーム分けアプリは、現在のチーム分け結果を画像形式(PNG または JPEG)に変換すること
3. 画像変換後、チーム分けアプリは、その画像をユーザーの端末のローカルストレージにダウンロードすること
4. 保存される画像には、全てのチームとメンバー情報が視認可能な状態で含まれること

### 要件8: UI/UXデザイン

**目的:** プロジェクトチームのユーザーとして、シンプルで使いやすいインターフェースを利用したい。そうすることで、迷わず素早くチーム分けを実行できる。

#### 受入基準
1. チーム分けアプリは、シンプルでミニマルなデザインを採用すること
2. チーム分けアプリは、青色をメインカラーとして使用すること
3. チーム分け実行時、チーム分けアプリは、複雑なアニメーションを使用せず、結果を即座に表示すること
4. チーム分けアプリは、主要な操作をワンクリックで実行可能にすること

### 要件9: デプロイメント

**目的:** プロジェクトチームのユーザーとして、インターネット経由でアプリケーションにアクセスしたい。そうすることで、どこからでもチーム分けを実行できる。

#### 受入基準
1. チーム分けアプリは、GitHub Pagesでホスティングされること
2. チーム分けアプリは、Webブラウザからアクセス可能であること
3. チーム分けアプリは、アカウント登録なしでゲストとして利用可能であること
4. インターネット接続がない場合、チーム分けアプリは利用できないこと

### 要件10: データ管理

**目的:** プロジェクトチームのユーザーとして、データ管理を気にせずアプリを利用したい。そうすることで、個人情報保護やデータ管理の負担なく使用できる。

#### 受入基準
1. チーム分けアプリは、ユーザーデータをサーバーに保存しないこと
2. チーム分けアプリは、メンバー情報をセッション中のみブラウザ内に保持すること
3. ユーザーが明示的に画像保存を実行した場合のみ、チーム分けアプリは、データをローカルストレージに保存すること
4. チーム分けアプリは、無料で提供されること
