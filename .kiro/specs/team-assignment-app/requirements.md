アプリケーション仕様書:チーム分けアプリ

1. アプリ概要 (Overview)
   アプリ名(仮): チーム分けアプリ for NAiS
   コンセプト: 自チームのチーム分け制約に合わせて、ランダムなチーム分けがワンクリックでできるアプリ
   ターゲットユーザー: プロジェクトチームのみ
   対応プラットフォーム: Web アプリ(github pages)
   課金モデル: 無料

2. ユーザーフロー (User Flow)
   ユーザーがアプリを開いてから目的を達成するまでの流れ。
   メンバー登録: 以下のメンバーが自動で入力されている
   ひらがなメンバー:5 人(あ〜お)
   カタカナメンバー:5 人(ア〜オ)
   条件設定: [ チーム数指定 / 1 チームの人数指定 / 1 チームのみのひらがな or カタカナメンバー数指定 ]
   チーム分け実行: [ 即時表示 ]
   結果の共有: 画像保存

3. 機能要件 (Functional Requirements)
   3-1. メンバー管理機能
   入力項目:
   名前 [ 必須 ]

   基本ルール:
   [ A: チーム数を指定して分ける(例:3 チームに分ける) ]
   [ B: 1 チームの人数を指定して分ける(例:4 人ずつに分ける) ]
   制約を指定して分ける(例:1 チームは NAiS メンバーのみにして分ける)
   均等化オプション(バランス調整):
   なし。ランダムにチーム分けができれば良い。
   端数の処理: 人数が割り切れない場合、ランダムに配分
   3-3. 結果表示・編集機能
   表示形式: [ リスト表示 / カード形式 ]
   再抽選: 結果が気に入らない場合、[ ワンタップでやり直し可能 ]
   タイマー機能: チーム分け後にゲーム時間を計測する機能は [ 不要 ]
   スコア記録: 勝敗や得点を記録する機能は [ 不要 ]
